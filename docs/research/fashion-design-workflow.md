# 패션 디자인실 워크플로우 리서치 — 한국 영캐주얼 RTW+DTC 믹스 컨텍스트

> 27SS 컬렉션 작업을 진행 중인 한국 영캐주얼/디자이너 RTW+DTC 믹스 브랜드의 디자인실 실무 흐름을 정리한 리서치. 무신사·29CM 입점 중소 디자이너 브랜드(인사일런스, 디스이즈네버댓, 라이, 우알롱, 노매뉴얼, 노이스, 애즈온 등) 톤을 가정한다. 마케팅·룩북·세일즈는 다운스트림이며 이 문서의 범위 밖이다.

## 0. 리서치 노트

- **타겟 브랜드 톤**: 디자이너 색이 살아있되 무신사·29CM·W컨셉을 메인 채널로 운영하는 중소 RTW. 시즌당 약 30~80 SKU, 자사몰+편집몰 병행, 디자인실 인원 3~10명 규모(추정).
- **시즌**: 27SS. 통상 2026년 9월 전후 컨셉팅 시작 → 2027년 1~3월 출고가 표준 흐름이다 ([Heuritech](https://heuritech.com/articles/fashion-seasons-the-complete-guide/), [ApparelMagic](https://apparelmagic.com/understanding-the-fashion-calendar-in-2024-and-beyond/)).
- **목적**: 창작-개발 시퀀스(컨셉 → 소재 → 컬러 → 실루엣 → 디테일 → 샘플)를 정확히 잡고, 어떤 산출물이 도구화 가능한지 갭을 본다.
- **한계**: 한국 영캐주얼 디자이너 브랜드는 디자인실 내부 프로세스를 거의 공개하지 않는다. 따라서 일부 수치(디자인실 인원, 캐리오버 비율, 시즌별 SKU 등)는 업계 통설·간접 소스 기반 **추정**이며, 본문에 "(추정)"으로 명시한다.

## 1. 디자인실 워크플로우 베이스라인

### 1.1 시즌 캘린더 (SS/FW)

글로벌 패션 캘린더 통상은 SS·FW가 각각 출시 6~7개월 전 디자인 시작이지만, 영캐주얼 DTC는 더 빨리 압축된다. 이유: ① 무신사 시즌 프리뷰 같은 입점몰 사전 검증 일정에 맞춰야 하고, ② 동대문·국내 봉제 인프라 덕분에 리드타임이 글로벌보다 짧으며, ③ 자사몰 드롭이 매월 돌면서 시즌 경계가 흐려진다.

**27SS 표준 일정 (역산)**

| 시점 | 단계 | 주요 산출물 |
| --- | --- | --- |
| 2026.07~08 | 시즌 리서치·트렌드 인풋 | WGSN/Coloro 27SS 키컬러 입수, 글로벌 SS26 런웨이 분석, 자체 시장 리서치 |
| 2026.08~09 | 컨셉 빌드 | 컨셉보드, 키워드 셋, 무드보드, 컬러 팔레트 1차 |
| 2026.09~10 | 소재·컬러 개발 | 동대문/PV·PIS 소재 서치, 스왓치 1차, Pantone TCX 컬러카드 픽 |
| 2026.10~11 | 실루엣·도식화 | 라인업 시트 v1, 도식화(플랫) 60~120장, 패턴실 의뢰 |
| 2026.11~12 | 1차 샘플 (개발 샘플) | 1st 샘플 평가, 핏 보정, 트림 확정 |
| 2026.12~2027.01 | 2차 샘플·확정 | 2nd 샘플(컨펌 샘플), 최종 사양서, 컬러웨이 확정 |
| 2027.01 | 무신사 시즌 프리뷰·자사몰 티저 | 시즌 프리뷰(연 4회 개최), 프리오더 검증 ([Musinsa Newsroom](https://newsroom.musinsa.com/newsroom-menu/2024-0226-24fw-season-preview)) |
| 2027.02~03 | 본 생산 / 1차 출고 | PP 샘플, 메인오더 작업지시서, QC |
| 2027.03~07 | SS 드롭 (월 단위) | 1차 드롭(3월) → 2차(4~5월) → 3차(6~7월) |

핵심: SS와 FW는 **늘 6개월 겹친다**. 27SS 마무리(2027 1~3월) 시점에 27FW 컨셉팅이 이미 진행되며, 영캐주얼은 보통 SS·FW를 따로 얹기보다 SS 시즌 중간에도 캐리오버·캡슐 드롭을 끼워 넣는다 ([SupplyCompass — Fashion Drop](https://supplycompass.com/sustainable-fashion-blog/drops-fashion-production/)).

서울패션위크 2026 SS는 9월 1~7일 DDP 외 서울 전역 개최, 27개 디자이너 브랜드 참여 예정 ([서울시 보도자료](https://news.seoul.go.kr/economy/archives/568204)). 27SS는 통상 2026년 9월 진행 예상이지만, 영캐주얼 DTC 브랜드 다수는 패션위크에 직접 참여하지 않고 무신사 시즌 프리뷰·자사몰 캠페인으로 대체한다.

### 1.2 디자인 팀 구조와 R&R

한국 영캐주얼 중소 디자이너 브랜드는 보통 다음 직제가 압축된 형태로 돈다(추정 기반의 일반화):

| 롤 | 핵심 R&R | 자주 쓰는 도구 |
| --- | --- | --- |
| 크리에이티브 디렉터(=대표/오너 디자이너) | 시즌 컨셉·아이덴티티 결정, 최종 컨펌, 컬러·실루엣 톤 디렉팅 | Pinterest, 무드보드, 직접 일러스트, Notion |
| 수석/시니어 디자이너 | 시즌 라인업 빌드, 카테고리별 디자인 담당, 패턴실·샘플실과의 1차 커뮤니케이션 | Adobe Illustrator(도식화), Photoshop, WGSN |
| 디자이너 | 도식화·작업지시서 작성, 소재 서치, 컬러 픽, 1·2차 샘플 코멘트 | AI, 엑셀/구글시트, 사내 ERP, 카톡 |
| 어시스턴트 디자이너 | 스왓치 정리, 트림 서치, 작업지시서 보조 입력, 샘플 핏팅 보조, 시장 리서치 | 엑셀, Pinterest, 동대문 매장 사진 |
| 머천다이저(MD) | 라인업 카운트·가격 결정, 캐리오버 vs 신상 비율, 카테고리 밸런스, 출고 드롭 일정 | 엑셀, ERP, 매출 데이터 |
| 패턴사(인하우스 또는 외주) | 패턴 드래프팅, 핏 블록 관리, 그레이딩, 사이즈 스펙 | 옵티텍스/유카(Yuka)/CLO, 종이 패턴 |
| 샘플사 | 1st·컨펌·PP 샘플 봉제 | 산업 미싱, 오버록, 특수 미싱 |
| 생산팀(MD 겸직 多) | 봉제공장 발주·QC, 부자재 발주, 납기 관리 | 어바옷·오슬 등 매칭 플랫폼([abaot.co.kr](https://www.abaot.co.kr/), [osle.co.kr](https://osle.co.kr/)) |

영캐주얼 중소 브랜드의 현실: 디자이너 1명이 카테고리 디자인+도식화+소재 서치+샘플 코멘트+가끔 룩북 셀렉까지 다 한다. 패턴실·샘플실은 거의 외주이며, 결이 맞는 1~2개 업체와 장기적으로 일하는 구조가 효율적이다 ([@bone_pattern threads post](https://www.threads.com/@bone_pattern/post/DHu6fc3SKxq)).

### 1.3 마일스톤 시퀀스 (창작-개발)

```
시즌 리서치 → 컨셉팅 → 컬러 셋팅 → 소재 개발 → 실루엣 개발
   → 디테일 디자인 → 라인업 빌드 → 도식화/작업지시서 → 1st 샘플
   → 핏·소재 보정 → 컨펌 샘플 → PP 샘플 → 메인오더 → 출고
```

각 게이트별 산출물:

| 게이트 | 산출물 | 주 작성자 |
| --- | --- | --- |
| 시즌 리서치 | 트렌드 리포트, 경쟁 브랜드 분석, 자사 베스트셀러 회고 | 디렉터+MD |
| 컨셉팅 | 컨셉보드(키워드 5~10), 무드보드, 시즌 한 줄 스토리 | 디렉터 |
| 컬러 셋팅 | 컬러 스토리(메인+액센트), Pantone TCX 코드, 컬러칩 보드 | 디자이너+디렉터 |
| 소재 개발 | 스왓치북, 소재 카드, MOQ·리드타임 표 | 디자이너+소재담당 |

| 디테일 디자인 | 디테일 도식화, 트림 라이브러리 픽, 자수/프린트 시안 | 디자이너 |

### 1.4 글로벌 패션하우스 vs 한국 영캐주얼 RTW+DTC 차이

| 항목 | 글로벌 럭셔리(르메르·에르노·마르지엘라 톤) | 한국 영캐주얼 RTW+DTC |
| --- | --- | --- |
| 시즌 캘린더 | 출시 6~12개월 전 디자인 시작, 패션위크 6개월 전 공개 ([nss magazine](https://www.nssmag.com/en/fashion/22450/fashion-seasons-calendar)) | 출시 4~6개월 전 디자인 시작, 시즌 중에도 매달 드롭 |
| 소재 채널 | PV Paris·Milano Unica·이태리·일본 밀 직거래, 자체 개발 비중 큼 | 동대문 종합시장, PV Seoul·Preview in Seoul(PIS), [SwatchOn](https://swatchon.com/), 일부 이태리·중국 |
| MOQ | 50~500 야드 단위 직거래, 12~16주 리드타임 | 동대문 보유 원단은 사실상 MOQ 없음, 개발 원단은 50~100 야드 4~8주(추정) |
| 수직통합 vs 외주 | 자체 아틀리에·인하우스 패턴실·샘플실 비중 높음 | 패턴·샘플실은 거의 외주, 생산은 국내 봉제 + 일부 베트남·중국 |
| 패션위크 영향 | 비즈니스 코어; 바이어 수주가 매출의 큰 축 | 일부 디자이너 브랜드만 SFW 참여; 대부분은 무신사·29CM·자사몰이 수주처 |
| 컬렉션 단위 | 시즌 1회 메인 + 프리폴/리조트 + 콜라보 | 시즌 + 매월 드롭 + 캡슐(콜라보·아카이브 리이슈) |
| 가격 의사결정 | 컬렉션 코어 / 카니발리제이션 통제 | 입점몰 가격 트래픽·할인 압력에 영향받음 |
| 카니발 분석 | 시즌-시즌 비교, 셀-스루 데이터 | 자사몰+무신사 셀-스루, 리뷰 수, 재구매율 |

### 1.5 일상 도구·산출물 인벤토리

**도구**

- **Adobe Illustrator**: 도식화(플랫) 1순위. "브랜드 디자이너 대부분이 그리는 그림은 스타일화가 아니라 도식화" — 한국 디자이너 실무의 정설 ([브런치 09화](https://brunch.co.kr/@@5OQt/9)). 27SS 시즌이면 카테고리별 60~150장 분량.
- **Photoshop**: 무드보드, 컬러 보정, 프린트 시안, 자수 시안.
- **Pinterest**: 시즌 초기 무드 스크랩, 빠른 비주얼 시드. 디자이너 개인 보드+팀 비공개 보드 병행.
- **WGSN / Coloro**: 컬러·소재·실루엣·디테일 트렌드 인풋. 27SS 키컬러 발표는 2025년 4월 ([WGSN/Coloro press release](https://www.wgsn.com/en/wgsn/press/press-releases/wgsn-and-coloro-reveal-colour-year-2027-luminous-blue-and-s-s-27-key)).
- **Pantone TCX/TPG 북**: 원단용 TCX, 인쇄·프레젠 용 TPG. TCX는 면 위에 염색된 표준이라 실제 옷 컬러 매칭에 정확. TPG가 약 15% 더 밝게 보이며, 코스트는 TCX의 40~60% 수준 ([Adstronaut](https://www.adstronaut.net/blog/pantone-color-matching-fashion), [Trimsbest](https://www.trimsbest.com/pantone-tcx-vs-tpg-difference-explained-complete-guide/)).
- **PV Seoul / Preview in Seoul / Texworld / 인터텍스타일 상하이**: PIS는 KOFOTI 주관 한국 최대 소재전, 8월 개최(2026 PIS는 8.19~21 COEX) ([Preview in Seoul](https://previewinseoul.com/)).
- **Notion**: 시즌 위키, 작업 지시 트래킹. 중소 브랜드에서 점진 도입 중.
- **Figma**: 일부 영캐주얼 브랜드가 라인업 시트·라인업 보드 빌드용으로 사용(추정).
- **엑셀/구글 시트**: 라인업 시트·BOM·사이즈 스펙·소재 코스트 시트의 사실상 표준.
- **사내 ERP**: 발주·재고·정산. 중소 브랜드는 ERP 대신 시트+카톡 운영도 흔함.
- **CLO 3D**: 디지털 샘플링 도입 중인 브랜드들이 채택 ([CLO3D KR](https://clo3d.com/ko/)). 대부분 영캐주얼은 아직 본격 도입 전.
- **카카오톡**: 패턴실·샘플실·공장과의 실시간 커뮤니케이션 표준 채널. 사진 첨부+코멘트가 사실상의 작업지시서 보완.
- **CAD 패턴(유카/옵티텍스)**: 패턴실 외주가 운용. 디자이너는 결과만 받음.

### 2.3 캐리오버 vs 신상 비율

> "캐리오버는 시즌과 무관하게 연속 판매되는 아이템" — [어패럴뉴스](http://m.apparelnews.co.kr/news/news_view/?idx=190313).

영캐주얼 캐리오버 운영 패턴(추정):

- **신생 브랜드 (1~3 시즌)**: 캐리오버 0~10%, 신상 90%+. 아카이브가 부족.
- **안정기 브랜드 (4~10 시즌)**: 캐리오버 20~40%, 신상 60~80%. 베스트 데님·셔츠·니트가 영구 라인.
- **헤리티지 빌드 단계**: 캐리오버 30~50%까지. 시그니처 데님, 시그니처 트렌치, 시그니처 니트.

캐리오버를 늘리는 이유: 비수기 매출 안정화, 시그니처 형성, 마케팅 비용 절감. 일부 브랜드는 시그니처 파카·트렌치·데님을 캐리오버로 두고 매년 컬러·디테일만 갱신.

## 3. 소재 (Material / Fabric)

### 3.1 소재 개발 흐름

```
보유 스왓치북 리뷰 → 박람회/시장 서치 → 1차 스왓치 픽
   → 협력 밀에 컬러웨이/포멧 의뢰 → 랩딥(lab dip) → 컨펌
   → MOQ/리드타임 협상 → 시즌 발주
```

영캐주얼 RTW의 채널 현실:

1. **동대문 종합시장**: 사실상 모든 영캐주얼 브랜드의 1차 소싱처. 1969년 개장한 종합 상가, A~D동, 약 3천여 점포 ([나무위키 — 동대문종합시장](https://namu.wiki/w/%EB%8F%99%EB%8C%80%EB%AC%B8%20%ED%8C%A8%EC%85%98%ED%83%80%EC%9A%B4%20%EA%B4%80%EA%B4%91%ED%8A%B9%EA%B5%AC), [위키백과](https://ko.wikipedia.org/wiki/%EB%8F%99%EB%8C%80%EB%AC%B8%EC%A2%85%ED%95%A9%EC%8B%9C%EC%9E%A5)). 보유 원단은 MOQ 거의 없거나 야드 단위 가능 ([인사이터 — 동대문 의류 사업](https://insight-er.com/%EB%8F%99%EB%8C%80%EB%AC%B8-%EC%9D%98%EB%A5%98-%EC%82%AC%EC%97%85-a-to-z/)). 부자재(단추·지퍼·라벨)는 종합시장 1~5층, 신설동·평화시장 보조.
2. **DDP / 신설시장**: 보조 채널.
3. **Preview in Seoul (PIS)**: KOFOTI 주관 한국 최대 소재·트림·기계 박람회, 27회차(2026)가 8.19~21 COEX 개최 예정 ([Preview in Seoul 2026](https://previewinseoul.com/)).
4. **Premiere Vision Paris/Seoul, Texworld, Intertextile Shanghai**: 글로벌 소재 트렌드 인풋. 영캐주얼 중소 브랜드는 직접 출장보다 PV의 디지털 카탈로그·트렌드 인사이트 자료를 활용하는 빈도가 높다.
5. **SwatchOn / FabricTime**: 스타트업 패브릭타임이 운영하는 동대문 원단 디지털 플랫폼. 약 200,000+ 원단 등록, MOQ 1야드 시작 (커스텀 프린트 5야드), 3야드 MOQ 가능 ([SwatchOn](https://swatchon.com/), [KED Global](https://www.kedglobal.com/korean-startups/newsView/ked202205190020)). 글로벌 매출 비중이 80%지만 국내 영캐주얼 브랜드도 디지털 카탈로그·샘플 신청 용도로 활용.
6. **이태리·일본·중국 밀 직거래**: 영캐주얼 디자이너 브랜드는 시그니처 아이템(트렌치·울 코트·실크 셔츠)에 한해 이태리·일본 밀 소싱 시도. 다만 MOQ·언어·리드타임 부담으로 코어 컬렉션은 국내 위주.
7. **한국섬유개발연구원(KTDI), KOFOTI**: 정부·협회 지원으로 디지털 소재 라이브러리, 소재 개발 컨설팅 ([KOFOTI](https://koreafashion.org/info/info_content_view.asp?cataIdx=803)).

### 4.3 영캐주얼 컬러 결정 현실

> "패션색채는 기본색과 유행색으로 나뉜다. 기본색은 수년 동안 지속 인정되는 색." ([rjcc.or.kr](http://www.rjcc.or.kr/journal/article.php?code=17259))

영캐주얼은 **보험성과 공격성의 비율 조정 게임**:

- 화이트·아이보리·블랙·차콜·네이비·베이지·올리브: 시즌 무관 안전 컬러. 셀-스루의 60~80%를 캐리.
- 시즌 컬러: WGSN/Pantone 인풋 + 한국 시장 색감(파스텔, 더스티, 뮤트 톤 선호) + 인스타그램 #ootd 색감 분석.
- 한국 영캐주얼 특이점: 글로벌 트렌드의 채도를 한 단계 낮춰서 채택하는 경향. 27SS Pop Pink (Coloro 151-73-22) 그대로 쓰기보다는 더스티 핑크·머스크 핑크로 변형.
- **인스타그램 색감**: 카메라 보정·필터 기준에서 잘 살아나는 색감을 선호. 더스티·아이보리·파스텔 톤이 사진에서 안정적이라 영캐주얼이 광범위하게 채택.
- 2026 패션 트렌드는 뉴트럴에서 활기 있는 컬러로 이동, 레드 부각 ([W Korea](https://www.wkorea.com/2025/12/30/%EC%98%AC%ED%95%B4-%ED%8C%A8%EC%85%98-%EC%9A%B4%EC%84%B8-%EB%8B%B5%EC%9D%80-%EC%9D%B4/)). 2025 가을은 모카브라운·딥버건디·차콜그레이·클래식베이지·더스티블루·올리브가 한국 시장에서 강세였다.

### 4.5 Pantone TCX vs TPG

- **TCX (Textile Cotton eXtended)**: 100% 면 위 염색, 2,800+ 컬러. 실제 옷 컬러 매칭의 골드 스탠다드 ([Adstronaut](https://www.adstronaut.net/blog/pantone-color-matching-fashion)).
- **TPG (Textile Paper Green)**: 종이 위 컬러, 친환경 잉크, TCX 대비 약 15% 밝게 보임. 디자인 작업·프레젠 용 ([Trimsbest](https://www.trimsbest.com/pantone-tcx-vs-tpg-difference-explained-complete-guide/)).
- 작업지시서·랩딥은 반드시 TCX 코드 명시. TPG는 내부 컬러 스토리·룩북에 적합.

### 4.6 다잉 랏(Dyeing Lot) QC

같은 TCX 코드라도 염색 랏마다 미세한 색차이(△E)가 발생. 영캐주얼 표준 QC:

- **랩딥(Lab Dip) 컨펌**: 발주 전 원단 염색 작은 샘플로 색 컨펌. △E 1.5 이내 통상 합격.
- **벌크 컬러 매칭**: 메인 발주 후 첫 롤 컨펌, 라이트박스(D65) 아래 검수.
- **랏간 통제**: 같은 스타일은 1롯에서 다 자르거나, 2롯 이상이면 사이즈 단위로 분리하여 풀백/혼입 방지.

## 5. 실루엣 (Silhouette)

### 5.1 베이스 슬로퍼/블록 시스템

> "패턴블록(슬로퍼)은 브랜드 시그니처 핏의 백본." ([Ninghow Apparel](https://www.ninghowapparel.com/blog/block-patterns-101-basic-block-sloper-master-pattern-fit-consistency/))

영캐주얼 운영 패턴(추정):

- **신생 브랜드**: 외주 패턴실의 표준 블록 사용. 시즌마다 새로 드래프팅하는 비효율.
- **중기 브랜드 (3~5 시즌+)**: 자사 슬로퍼 라이브러리 구축. 셔츠 블록, 니트 블록, 데님 블록, 슬랙스 블록, 자켓 블록 정도가 핵심.
- **블록 구축의 효과**: 핏 컴플레인 30~50% 감소, 시즌간 일관성 확보, 시즌 작업 시간 단축.

### 5.2 시즌 실루엣 키워드

영캐주얼 시즌 실루엣 키워드는 보통 5~8개 카테고리 단위:

| 키워드 | 설명 | 예시 카테고리 |
| --- | --- | --- |
| Boxy | 박시 핏, 어깨 직선, 길이 짧음 | 셔츠, 자켓 |
| Dropped Shoulder | 어깨 떨어짐, 캐주얼 무드 | 셔츠, 니트, 자켓 |
| Voluminous | 볼륨감, 풍성한 슬리브·헴 | 드레스, 블라우스 |
| A-line | A 라인, 허리 → 헴 확장 | 스커트, 드레스 |
| Column | 직선 실루엣, 수직 강조 | 코트, 드레스 |
| Cocoon | 어깨 → 헴 좁아지는 둥근 형 | 코트, 자켓 |
| Cinched | 허리 강조, 벨트·다트 | 자켓, 드레스 |
| Deconstructed | 비대칭·언피니시 | 셔츠, 자켓(디자이너성 강조) |
| Wide-leg | 넓은 보텀, 80s 회귀 | 데님, 슬랙스 |

### 5.3 25SS / 26SS 글로벌 트렌드 인풋

- **SS25**: anatomical distortion, surrealist silhouette, 풍선 어깨, 인플레이트 힙, 파워 푸프 드레스 ([The Glossary Magazine](https://theglossarymagazine.com/fashion/ss25-fashion-trends/)).
- **SS26**: 80s 회귀, 강한 어깨·시츠드 웨이스트 파워 드레싱, 드레이프드 스커트, 글로시·새틴, 노트·보우, 3D 플로럴, 버블 헴 ([Divo Boutique](https://www.divoboutique.com/en-us/blogs/news/spring-summer-2026-fashion-trends-what-changes), [FashionUnited — bubble hem](https://fashionunited.com/news/fashion/ss26-trend-designers-use-the-bubble-hem-to-create-a-fresh-silhouette/2025111469084), [Checkpoint Systems](https://checkpointsystems.com/blog/the-top-5-ss26-fashion-trends/)).
- **SS27 (예측)**: WGSN의 "interconnectedness" 테마와 SS26의 80s 무드를 결합한 더 정제된 파워 라인이 부상할 가능성(추정). 다만 영캐주얼 한국 시장은 트렌드를 1~2 시즌 늦게 받아들이는 경향.

### 5.4 영캐주얼 자주 쓰는 실루엣

- **셔츠**: Boxy + Dropped Shoulder가 최근 4~6 시즌 표준.
- **니트**: 라이트 게이지 카디건의 슬림+크롭, 또는 오버사이즈 라운드.
- **데님**: 와이드·세미와이드·스트레이트가 수년째 강세, 스키니는 거의 사라짐.
- **자켓**: 셔츠 자켓·블레이저는 박시·살짝 오버. 트렌치는 클래식 컷 유지 + 컬러 변형으로 차별화.
- **드레스**: 미디 + A-line 또는 Column. SS는 슬리브리스·오프숄더 비중 증가.

### 6.5 "영캐주얼은 디테일로 시즌감 표현" 검증

가설: 영캐주얼은 컬러·실루엣은 보수적이고 디테일이 차별화 포인트인가?

관찰(추정·간접 근거):

- **부분적 참**: 시그니처 셔츠/티셔츠는 실루엣·컬러 변형보다 칼라·헴·자수·프린트·트림 변형으로 시즌감을 만든다. 디스이즈네버댓 SS25 "Airport Style II"의 차별화 요소가 워시 기법·자카드·카모 패턴·시티 프린트였던 점은 이 가설을 뒷받침 ([Hypebeast — TINT SS25](https://hypebeast.kr/2025/2/thisisneverthat-ss25-spring-summer-2025-collection-lookbook-release-info)).
- **반례**: 디자이너 시그니처가 강한 라이(LIE)·노매뉴얼은 실루엣·소재 자체로 시즌감을 만든다.
- **결론**: 영캐주얼 매스 마켓일수록 디테일 의존도 높음, 디자이너성 강한 컨템포러리일수록 실루엣·소재 의존도 높음.

### 6.6 디테일 사양서 형태

도식화 + 디테일 클로즈업 + 사양 표가 1세트로 묶이는 게 표준. 한 스타일당 보통 2~4페이지:

```
페이지 1 — 도식화(앞/뒤) + 사이즈 스펙(주요 치수 표)
페이지 2 — 디테일 클로즈업(칼라·소매·포켓·헴) + 봉제 사양
페이지 3 — BOM (메인/서브/안감/트림/스티치 사양)
페이지 4 — 컬러웨이별 도식화 + 라벨 위치
```

도식화 작성 표준(한국 디자이너 양식):

- 옷의 형태·디테일·봉제 방법을 1:1 비례에 가깝게.
- 앞/뒤 + 디테일 클로즈업.
- 봉제 라인은 실선/점선으로 구분, 스티치 길이·폭은 글자로 명기.
- 사이즈 스펙은 어깨너비·가슴둘레·소매길이·총장·헴둘레·소매부리·암홀·넥라인 너비·길이 등을 mm 단위로.
- 컬러웨이별 도식화는 별도 시트 또는 컬러칩 첨부.

작업지시서 표준 양식의 BOM 항목:

```
| Item     | Description       | Composition | Color | Qty/PC | Supplier | Code      |
| Main     | Cotton Twill 220g | 100% Cotton | Ivory | 1.8 yd | Mill A   | FAB-038   |
| Lining   | Bemberg           | 100% Cupro  | Ivory | 0.9 yd | Mill B   | FAB-091   |
| Button   | 4-hole 18L        | Corozo      | Match | 8 pc   | Btn A    | BTN-127   |
| Zipper   | YKK 5VS Open      | -           | Match | 1 pc   | YKK      | ZIP-YKK-5 |
| Label    | Main Woven        | -           | Black | 1 pc   | Label A  | LBL-MN    |
```

## 7. 갭 분석 — 현재 페인 포인트와 도구화 기회

### 7.1 단계별 페인 포인트 인벤토리

| 단계 | 페인 포인트(현실) | 빈도 | 페인 강도 |
| --- | --- | --- | --- |
| 컨셉팅 | Pinterest·핸드폰 캡쳐가 흩어짐. 무드보드 만들 때 PSD에서 수동 콜라주, 매번 새로 시작 | 시즌 1~3회 | 중 |
| 컬러 셋팅 | TCX/TPG 칩 보드는 손으로 자르고 붙임. 디지털 컬러 스토리는 매번 PSD/AI로 재작업 | 시즌 1~5회 | 상 |
| 소재 개발 | 스왓치 정보가 카톡·엑셀·실물 카드에 분산. 같은 소재를 두 번 픽하거나, 시즌 끝나면 어디 있는지 모름 | 시즌 30~80회 | 매우 상 |
| 라인업 빌드 | 엑셀 시트 vs 일러스트 라인업 보드 vs MD 시트가 따로 굴러감. 버전 동기화 어려움 | 시즌 3~8회 갱신 | 매우 상 |
| 도식화/사양서 | AI 파일이 사람마다 양식이 다름. 같은 도식화의 컬러웨이만 바꾼 버전이 시간 잡아먹음 | 스타일당 1세트 × 60~150 | 매우 상 |
| 샘플 평가 | 종이 시트나 카톡 사진+멘트로 흘러감. 1차→2차→PP 변경사항 추적이 노션에도 잘 안 정리됨 | 1차/2차/PP × 60~150 | 상 |
| 트림 라이브러리 | 박스 + 엑셀이 표준. 시즌 끝나면 트림 인덱스 갱신을 못함 | 상시 | 중 |
| 카니발리제이션 회고 | 시즌 데이터를 모으는 자체가 노가다. ERP·셀-스루·리뷰·재구매를 한 시트에 모으기 어려움 | 시즌 1~2회 | 상 |

### 7.2 페인 × 빈도 매트릭스 (도구화 우선순위)

```
              빈도 ↑
                |
      소재카드  | 도식화/사양서
      라인업   | 컬러스토리
                |
   샘플평가   | 카니발 회고
   트림인덱스 | 컨셉/무드보드
                |
              ─────────→ 페인 강도
```

상위 우선순위(높은 빈도 + 높은 페인):

1. **도식화/사양서(Tech Pack) 자동 양식 + 컬러웨이 자동 변형**: 빈도 압도적, 페인 큼.
2. **소재 스왓치 카드 시트화 + 디지털 인덱스**: 빈도 큼, 페인 매우 큼.
3. **라인업 시트(스타일×컬러웨이×가격×드롭)**: 빈도 큼, 페인 큼, 의사결정 핵심 도큐먼트.
4. **컬러 스토리 시트**: 시즌 1~5회지만 형식 표준화 효과 큼, 룩북·작업지시서·바이어 시트와 자산 공유.

### 7.3 AI/도구화 가능 영역

`fashion-lookbook` 스킬이 시즌 룩북을 다룬다면, 그 **상류**에서 디자인실이 시즌 동안 만드는 도큐먼트 6종을 단일 HTML 산출물 + ima2 이미지 생성으로 도구화 가능:

| 후보 | 기존 fashion-lookbook과의 관계 | ima2 도움 |
| --- | --- | --- |
| 컨셉/무드보드 | 상류 (시즌 시작) | 매우 큼 (mood 이미지 생성) |
| 컬러 스토리 시트 | 평행 (룩북도 컬러 사용) | 작음 (컬러칩은 코드 기반) |
| 소재 스왓치 시트 | 평행 (룩북도 소재 노트) | 중간 (텍스처 시각화) |
| 실루엣 시리즈 시트 | 상류 (룩북 전 단계) | 매우 큼 (실루엣 시안) |
| 디테일 사양서 / 도식화 시트 | 평행 (작업지시서 라인) | 중간 (디테일 클로즈업 시안) |
| 카테고리 매트릭스 / 라인업 시트 | 상류 (룩북 빌드 인풋) | 작음 (구조적 시트) |

핵심 차별점: 영캐주얼 디자인실의 모든 도큐먼트는 본질적으로 **시각 + 표 + 코드**의 혼합이며, 단일 HTML 형식이 자연스럽게 그 모양을 흡수한다.

## 8. 스킬 후보 정리 (Phase 4)

### 8.1 fashion-concept-board

- **무엇을 산출**: 시즌 컨셉/무드 보드를 단일 HTML로. 키워드, 시즌 한 줄 스토리, 무드 이미지 9~16장, 컬러 시드, 소재 키워드, 실루엣 키워드.
- **핵심 섹션**: 커버(시즌·브랜드·컨셉문) → 키워드 클라우드 → 무드 그리드(ima2) → 컬러 시드 → 소재 키워드 → 실루엣 키워드 → 시즌 톤 다이얼.
- **입력**: 시즌(27SS), 브랜드 톤, 컨셉 키워드 3~7개, 레퍼런스 이미지 또는 무드 묘사.
- **fashion-lookbook과의 관계**: **상류**. 컨셉보드 산출물이 룩북의 시즌 톤·이미지 무드를 결정.
- **ima2**: 매우 도움. 시즌 이미지 9~16장 생성이 핵심.
- **모드**: prototype.

### 8.2 fashion-color-story

- **무엇을 산출**: 시즌 컬러 팔레트와 컬러웨이 분배 가이드를 단일 HTML로. Pantone TCX/TPG 코드 + 적용 비율 + 카테고리별 분배.
- **핵심 섹션**: 커버 → 시즌 컨셉 → 메인/시즌/액센트 컬러 그리드 → TCX·Coloro·HEX 코드 표 → 카테고리별 컬러웨이 분배 → 60/30/10 적용 가이드 → 다잉 랏 QC 노트.
- **입력**: 시즌, 컬러 키워드/시드, 카테고리 구성, 베이직 vs 시즌 비율.
- **fashion-lookbook과의 관계**: **평행**. 룩북의 컬러 노트와 자산 공유.
- **ima2**: 작음. 컬러칩 자체는 CSS로 충분, 무드 이미지 1~2장 보조 정도.
- **모드**: deck (정보 밀도 + 인쇄 가능).

### 8.3 fashion-material-swatch

- **무엇을 산출**: 시즌 소재 스왓치 시트를 단일 HTML로. 소재 카드 30~80장 + 검색·필터.
- **핵심 섹션**: 커버 → 소재 카테고리 인덱스 → 카드 그리드(코드/명/조성/중량/폭/가격/공급처/MOQ/리드타임/컬러웨이) → 메인-서브-트림 매칭 차트 → 시즌 소재 키워드.
- **입력**: 시즌, 소재 카드 데이터(엑셀/CSV 인포트 가능), 카테고리별 추천.
- **fashion-lookbook과의 관계**: **평행**. 룩북 소재 노트의 데이터 소스.
- **ima2**: 중간. 텍스처 시각화 보조 가능, 단 실물 스왓치 사진이 더 정확.
- **모드**: template (구조적 데이터 시트).

### 8.4 fashion-silhouette-board

- **무엇을 산출**: 시즌 실루엣 시리즈 시트를 단일 HTML로. 카테고리별 실루엣 키워드 + 도식화 시드 + 시즌 핏 가이드.
- **핵심 섹션**: 커버 → 시즌 실루엣 키워드(5~8개) → 카테고리별 시리즈(셔츠/니트/데님/자켓/드레스 등) → 핏 블록 메모 → 25SS·26SS·27SS 트렌드 비교.
- **입력**: 시즌, 실루엣 키워드, 카테고리 리스트, 자사 시그니처 핏.
- **fashion-lookbook과의 관계**: **상류**. 룩북 룩 단위의 실루엣을 정의.
- **ima2**: 매우 큼. 실루엣 시안 12~24장 생성.
- **모드**: prototype.

### 8.5 fashion-detail-spec (도식화·사양서)

- **무엇을 산출**: 스타일 단위 작업지시서(Tech Pack)를 단일 HTML로. 도식화(이미지) + BOM + 사양표 + 사이즈 스펙 + 컬러웨이.
- **핵심 섹션**: 헤더(시즌·브랜드·스타일코드) → 도식화 4면 → 디테일 클로즈업 → BOM 표 → 사이즈 스펙 표 → 봉제 사양 → 컬러웨이 그리드 → 라벨 위치.
- **입력**: 스타일 코드, 카테고리, 도식화 이미지(또는 ima2 시안), BOM 데이터, 사이즈 스펙.
- **fashion-lookbook과의 관계**: **평행/하류**. 룩북에서 픽한 룩의 사양화.
- **ima2**: 중간. 도식화는 정확도가 핵심이라 ima2는 시안용. 실제는 AI 파일 임베드.
- **모드**: template (인쇄·공장 발송 호환).

### 8.6 fashion-lineup-matrix (카테고리 매트릭스/라인업 시트)

- **무엇을 산출**: 시즌 라인업 시트를 단일 HTML로. 카테고리 × 스타일 × 컬러웨이 × 가격 × 드롭 매트릭스 + 캐리오버 표시 + 셀-스루 회고.
- **핵심 섹션**: 시즌 KPI(스타일 수, 컬러웨이 수, 캐리오버 비율, 평균 가격) → 카테고리 매트릭스(아우터/톱/니트/드레스/보텀/액세서리 등) → 스타일 행(코드/이미지/컬러웨이/가격/드롭) → 캐리오버 vs 신상 분포 → 가격 분포 차트.
- **입력**: 시즌, 스타일 데이터(CSV 인포트), 카테고리 정의, 가격 정책, 드롭 일정.
- **fashion-lookbook과의 관계**: **상류**. 룩북에서 보여줄 룩 단위의 의사결정 시트. 룩북의 line-up 섹션이 이 시트의 압축본.
- **ima2**: 작음. 스타일별 썸네일 시드용 정도.
- **모드**: deck/template 혼합.

### 8.7 fashion-sample-review (옵션)

- **무엇을 산출**: 1차/2차/PP 샘플 평가 시트를 단일 HTML로.
- **핵심 섹션**: 스타일 헤더 → 샘플 사진 슬롯(전·후·디테일) → 핏 코멘트 → 소재 코멘트 → 트림 코멘트 → 컬러 코멘트 → 다음 샘플 변경사항.
- **입력**: 스타일 코드, 샘플 차수, 사진(업로드), 평가 항목.
- **fashion-lookbook과의 관계**: 무관(개발 트랙).
- **ima2**: 작음. 사진은 실물 위주.
- **모드**: template.

### 8.8 우선순위 권고 (도구화 로드맵)

| 우선순위 | 스킬 | 이유 |
| --- | --- | --- |
| 1 | fashion-concept-board | 시즌 시작점. ima2 활용 가치 매우 큼. 룩북 상류로 연결성 높음. |
| 2 | fashion-color-story | 시즌마다 반드시 만들고, 형식 표준화 효과가 즉각적. |
| 3 | fashion-lineup-matrix | 디자이너+MD 합의 도큐먼트, 페인 매우 큼, 의사결정 도큐먼트. |
| 4 | fashion-silhouette-board | ima2 활용 가치 큼, 룩북 상류로 자연스럽게 연결. |
| 5 | fashion-material-swatch | 데이터 시트 성격, 단계적 개선 가능. |
| 6 | fashion-detail-spec | 정확도 요구가 높아 ima2가 시안 수준에 머무름. AI 파일 통합이 관건. |
| 7 | fashion-sample-review | 워크플로우의 일부지만 다른 스킬과 종속성 약함. |
