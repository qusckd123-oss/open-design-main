# 리오더 근거 이미지

리오더 의사결정자가 한 장만 보고 “왜 이 물량을 지금 넣어야 하는지” 판단할 수 있게 만드는 근거 이미지 제작 스킬이다. 분석 자체보다 승인용 시각화가 목적이므로 숫자, 그래프, 문장 근거를 한 화면 안에 일관되게 배치한다.

## 사용 상황

다음 요청이면 이 스킬을 사용한다.

- 특정 품번/컬러의 리오더 진행 근거를 이미지로 만들어야 할 때
- 엑셀 리오더 점검 파일에서 품번별 A4 1장 또는 여러 장 리포트를 만들어야 할 때
- 1차/2차 리오더 수량, 입고 예정일, 예상 소진일을 함께 보여줘야 할 때
- 사이즈별 예상 리오더 수량을 표로 넣어야 할 때
- MD/생산/대표 승인용으로 공유 가능한 PNG, PDF, XLSX가 필요할 때

`demand-optimizer`는 리오더 후보와 수량을 판단하는 분석 엔진이고, 이 스킬은 그 판단을 설득 가능한 이미지 자료로 포장하는 산출물 엔진이다. 판단 근거가 부족하면 먼저 `demand-optimizer` 방식으로 STR, 판매 속도, 소진 예상, 리드타임을 계산한 뒤 이 스킬로 시각화한다.

## 입력 확인

작업 전에 가능한 범위에서 아래 정보를 확인한다. 파일에서 찾을 수 있으면 따로 묻지 말고 추출한다.

| 항목 | 필요성 |
| --- | --- |
| 시즌, 브랜드, 품번, 컬러 | 제목과 파일명에 사용 |
| 초도 입고, 현재 판매, 현재 재고 | 핵심 KPI와 판매율 계산 |
| 주차별 판매량 | 막대 그래프와 판매 속도 산정 |
| 주차별 입고/리오더 이벤트 | 1차/2차 리오더 라벨 표시 |
| 사이즈별 판매/재고/소진율 | 사이즈별 리오더 수량 산정 |
| 리오더 수량과 입고 예정일 | 승인 요청의 핵심 결론 |
| 예상 소진 주차 또는 시즌 종료일 | 리오더 필요성 검증 |
| 온라인/오프라인/대마 물량 포함 여부 | 수량 기준의 오해 방지 |

필수 데이터가 누락되면 임의로 채우지 말고 “확정값”, “가정값”, “파일에서 미확인”을 구분해서 표시한다. 승인용 자료에서는 가정값이 결론처럼 보이면 안 된다.

## 산출물 원칙

1. A4 가로형 1페이지를 기본으로 만든다. 품번이 여러 개면 품번별 1페이지씩 만들고, 필요하면 통합 PDF를 추가한다.
2. 화면 첫 줄에 결론을 둔다. 예: `리오더 10,000 pcs 진행 권장`, `1차 3,000 pcs + 2차 7,000 pcs 반영 시 10/4 주차 소진`.
3. 숫자 단위는 `pcs`, `원`, `%`, `주차`로 통일한다. 천 단위 쉼표를 사용한다.
4. 그래프는 주차별 판매량 막대와 누적 판매/공급/기말 재고 선을 함께 보여준다.
5. 사이즈별 수량표는 총합이 리오더 총량과 일치해야 한다.
6. “왜 지금 리오더인가”를 한 문단으로 적는다. 판매율, 잔여 재고, 소진 예상, 리드타임을 연결한다.
7. 이미지 안의 한국어가 깨지지 않게 Windows에서는 `Malgun Gothic`, macOS에서는 `Apple SD Gothic Neo`, 웹에서는 `Noto Sans KR` 계열을 우선 사용한다.

## 권장 페이지 구성

```text
[헤더]
시즌/브랜드 리오더 진행 근거
품번 컬러 / 작성일 / 기준 물량 포함 여부

[결론 KPI 3칸]
리오더 권장 수량 | 입고 계획 | 승인 판단

[메인 그래프]
주차별 판매량: 막대
기말 재고: 빨간 선
누적 공급: 파란 점선
누적 판매: 검정 선
리오더 이벤트: 세로 점선 + 라벨
예상 소진일: 세로 점선 + 라벨

[오른쪽 요약표]
초도 입고, 현재 판매, 현재 재고, 현재 판매율, 리오더 후 총 운영 공급, 예상 소진 주차

[사이즈별 표]
XS/S/M/L/XL/XXL/합계별 권장 리오더 수량

[근거 문장]
판매 흐름, 재고 소진, 리드타임, 사이즈 편차를 연결한 승인용 문장
```

## 계산 기준

### 기본 KPI

```text
현재 판매율 = 현재 판매 / 현재 입고
리오더 후 총 운영 공급 = 현재 입고 + 확정 리오더 + 제안 리오더
예상 잔여 재고 = 누적 공급 - 누적 판매
예상 소진일 = 예상 잔여 재고가 0 이하가 되는 첫 주차
주차별 판매 속도 = 최근 N주 판매 평균 또는 유사 품번 주차 흐름 보정값
```

데이터가 충분하면 최근 4주, 8주, 전체 평균을 비교하고, 승인용 이미지에는 최종 채택한 기준만 노출한다. 보정이 들어가면 “전년 유사 스타일 흐름 기준”, “최근 4주 평균 기준”, “대마 물량 제외 기준”처럼 근거를 명시한다.

### 사이즈별 리오더 수량

1. 현재 판매 비중과 사이즈별 재고 부족도를 함께 본다.
2. 특정 사이즈 재고가 과다하면 단순 판매 비중대로 추가하지 않는다.
3. 최종 사이즈별 수량 합계는 리오더 총량과 정확히 일치시킨다.
4. 반올림 보정은 판매 비중이 큰 사이즈부터 배분한다.

```text
기본 배분 = 리오더 총량 * 사이즈별 판매 비중
부족 보정 = max(목표 잔여 재고 - 현재 재고, 0)
최종 배분 = 기본 배분과 부족 보정을 혼합한 뒤 총량에 맞게 보정
```

## 시각 스타일

- 배경은 흰색 또는 옅은 웜 그레이를 사용한다.
- 승인 판단은 짧고 명확하게 쓴다. 예: `OK`, `검토`, `보류`.
- 중요한 리오더 수량과 소진일은 굵게 처리한다.
- 그래프 색상 권장:
  - 주차별 판매량: muted teal
  - 기말 재고: red
  - 누적 공급: navy dashed
  - 누적 판매: charcoal
  - 이벤트 라벨: warm beige box
- 장식보다 판독성을 우선한다. 카드형 UI를 과하게 늘리지 말고, 표와 그래프의 정렬을 맞춘다.

## 파일 제작 절차

1. 원본 엑셀을 읽고 품번/컬러별 입력값을 정리한다.
2. 데이터 기준을 확정한다. 특히 대마 물량 포함/제외, 리오더 확정분/제안분을 분리한다.
3. 주차별 판매/재고/공급/누적 판매 테이블을 만든다.
4. 예상 소진 주차와 사이즈별 리오더 권장 수량을 계산한다.
5. A4 가로 HTML을 만든다. 필요하면 Chart.js, SVG, matplotlib 이미지 중 가장 안정적인 방식을 사용한다.
6. HTML을 PNG로 렌더링한다. Playwright 또는 브라우저 캡처가 가능하면 실제 렌더를 확인한다.
7. XLSX가 필요하면 같은 숫자와 그래프를 엑셀에 넣는다.
8. 마지막에 모든 합계와 그래프 라벨을 검산한다.

## 검산 체크리스트

- [ ] 현재 판매율 = 현재 판매 / 현재 입고이 맞는가?
- [ ] 총 리오더 수량 = 사이즈별 리오더 수량 합계인가?
- [ ] 리오더 후 총 운영 공급 = 현재 입고 + 리오더 확정/제안 수량인가?
- [ ] 그래프의 리오더 이벤트 날짜와 표의 입고 예정일이 같은가?
- [ ] 예상 소진 주차가 그래프와 문장에 동일하게 표시되는가?
- [ ] 대마 물량 포함/제외 기준이 헤더 또는 근거 문장에 표시되는가?
- [ ] 파일명에 품번/컬러가 들어가고 공백/특수문자가 안전하게 정리됐는가?
- [ ] PNG에서 한국어 글꼴이 깨지지 않는가?

## 좋은 결과의 기준

좋은 산출물은 예쁘기만 한 이미지가 아니라 결재자가 숫자를 다시 계산하지 않아도 되는 이미지다. 제목에서 결론을 이해하고, 그래프에서 재고가 왜 부족해지는지 보고, 표에서 어떤 사이즈를 얼마나 넣을지 확인할 수 있어야 한다.
